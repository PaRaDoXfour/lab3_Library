# Архітектура системи Library_project

## Компоненти та їх взаємодія
Проєкт побудований за принципом шаруватої архітектури:

1. **Presentation Layer (`Driver.java`)**: Консольний інтерфейс користувача. Обробляє ввід та виводить результати.
2. **Service Layer (`Library.java`)**: Містить бізнес-логіку керування фондом. Не знає про спосіб збереження даних.
3. **Data Access Layer (`FileHandler.java`)**: Відповідає за серіалізацію та десеріалізацію даних у формат `.txt`.



## Бізнес-логіка
- **Поліморфне копіювання**: При клонуванні бібліотеки використовується механізм Deep Copy для уникнення Side-effects.
- **Обробка типів**: Система розрізняє `RareBook`, `EBook` та `PhysicalBook`, застосовуючи до них різні правила валідації.
