# 🚀 Deployment Guide (Інструкція з розгортання)

Цей документ описує процес розгортання `Lab4_Library` у виробничому середовищі.

## 🖥 Вимоги до апаратного забезпечення

* **Архітектура:** x86_64 або ARM64.
* **CPU:** Мінімум 1 Core (2.0 GHz+).
* **RAM:** Мінімум 512 MB (рекомендовано 1 GB для стабільної роботи JVM).
* **Disk:** Мінімум 100 MB вільного місця (для логів та файлу бази даних `input.txt`).

## 📦 Необхідне програмне забезпечення

* **JRE 17 (Java Runtime Environment):** Виробниче середовище не потребує повного JDK, достатньо лише JRE.
* **Screen або Systemd:** (для Linux-серверів) для забезпечення фонової роботи консольного застосунку.

## 🌐 Налаштування мережі

Оскільки застосунок є консольним і не має мережевого інтерфейсу (HTTP/gRPC), спеціальне налаштування Firewall не потрібне, окрім доступу по SSH для адміністратора.

## 🔧 Конфігурація та розгортання

### 1. Підготовка артефакту
На машині розробника або CI/CD сервері зберіть проект:
```bash
mvn clean package
```
Це створить файл `Lab4_Library-1.0-SNAPSHOT.jar` у теці `target/`.

### 2. Розгортання коду
Скопіюйте артефакт на сервер:
```bash
scp target/Lab4_Library-1.0-SNAPSHOT.jar user@server:/opt/library-app/
```

### 3. Налаштування СУБД (Persistence)
* Переконайтеся, що у користувача, який запускає застосунок, є права на **запис/читання** у директорії `/opt/library-app/`.
* Файл `input.txt` повинен бути захищений від випадкового видалення іншими користувачами (права `644`).

### 4. Запуск у Production
Для запуску на сервері використовуйте команду:
```bash
java -jar Lab4_Library-1.0-SNAPSHOT.jar
```
Рекомендація: використовуйте `tmux` або `nohup`, щоб процес не завершувався після закриття SSH-сесії.

## ✅ Перевірка працездатності (Health Check)

Як зрозуміти, що все працює:

* **Логи запуску:** При старті в консолі має з'явитися головне меню бібліотеки.
* **Перевірка файлу:** Після виконання будь-якої операції видачі книги, перевірте, чи оновився час модифікації файлу `input.txt`.
* **Процеси:** Команда `ps aux | grep java` повинна відображати активний процес вашого JAR-файлу.
